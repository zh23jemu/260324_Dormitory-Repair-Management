# 接口设计清单

本文档给出高校宿舍报修管理系统的接口设计草案，适用于当前 `Spring Boot + Vue + SQLite` 的简化实现方案。

## 1. 统一约定

- 接口前缀建议：`/api`
- 返回格式建议统一为：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- 认证方式：登录成功后返回 token，前端通过请求头传递
- 请求头建议：

```http
Authorization: Bearer {token}
```

## 2. 认证接口

### 2.1 登录

- `POST /api/auth/login`

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

返回字段建议：

- `token`
- `userInfo`
- `role`

### 2.2 学生注册

- `POST /api/auth/register`

请求字段建议：

- `username`
- `password`
- `realName`
- `phone`
- `studentNo`
- `college`
- `major`
- `className`

### 2.3 获取当前登录用户

- `GET /api/auth/me`

### 2.4 修改密码

- `PUT /api/auth/password`

### 2.5 重置密码

- `PUT /api/auth/reset-password/{userId}`

说明：

- 首版建议仅管理员支持重置密码

## 3. 学生端接口

### 3.1 获取首页公告列表

- `GET /api/student/announcements`

### 3.2 提交报修申请

- `POST /api/student/repair-orders`

请求字段建议：

- `repairTypeId`
- `title`
- `description`
- `expectTime`
- `buildingId`
- `roomId`
- `imagePaths`

### 3.3 获取我的报修列表

- `GET /api/student/repair-orders`

查询参数建议：

- `status`
- `pageNum`
- `pageSize`

### 3.4 获取报修详情

- `GET /api/student/repair-orders/{id}`

### 3.5 提交评价

- `POST /api/student/repair-orders/{id}/rating`

请求字段建议：

- `score`
- `content`

### 3.6 获取个人信息

- `GET /api/student/profile`

### 3.7 更新个人信息

- `PUT /api/student/profile`

## 7. 通用上传接口

### 7.1 图片上传

- `POST /api/common/upload`

说明：

- 接收 `multipart/form-data`
- 返回文件访问路径

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "filePath": "/uploads/20260325/abc123.jpg"
  }
}
```

## 8. 建议优先实现顺序

建议按以下顺序开发接口：

1. `auth` 认证接口
2. 学生端报修申请、我的报修、评价接口
3. 宿管端审核、分配、工单查询接口
4. 维修人员接单、反馈接口
5. 公告接口
6. 宿舍管理接口
7. 管理员用户、统计、日志接口

## 9. 首版可裁剪项

如果要进一步压缩开发量，可将以下接口放到第二阶段：

- 字典管理接口
- 日志复杂筛选接口
- 宿舍信息导出接口
- 统计图表细分接口

首版至少应保证以下核心闭环：

- 登录
- 学生提交报修
- 宿管审核分配
- 维修人员接单反馈
- 学生评价
- 管理端查看工单列表
