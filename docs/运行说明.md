# 运行说明

## 环境要求

- `Java 17`
- `Maven 3.9+`
- `Node.js 24+`
- `npm 11+`

## 启动步骤

建议打开 3 个终端，分别启动后端、管理端、学生端。

### 1. 启动后端

```powershell
cd C:\Coding\260324_Dormitory-Repair-Management\server
mvn spring-boot:run
```

后端地址：

- `http://localhost:8080`

### 2. 启动管理端

```powershell
cd C:\Coding\260324_Dormitory-Repair-Management\admin-web
npm install
npm run dev
```

管理端地址：

- `http://localhost:5173`

### 3. 启动学生端

```powershell
cd C:\Coding\260324_Dormitory-Repair-Management\student-web
npm install
npm run dev
```

学生端地址：

- `http://localhost:5174`

## 一键启动

在项目根目录执行：

```powershell
cd C:\Coding\260324_Dormitory-Repair-Management
.\scripts\start-dev.ps1
```

该脚本会分别打开 3 个 PowerShell 窗口并启动三个服务。

## 示例数据说明

后端启动时会自动导入示例数据，包含以下内容：

- 多个用户账号
- 多个楼栋和宿舍
- 公告数据
- 不同状态的报修工单：
  - `待审核`
  - `已驳回`
  - `待接单`
  - `处理中`
  - `待评价`
  - `已完成`
- 维修结果记录
- 评价记录
- 系统日志

## 重新导入示例数据

本项目使用 `SQLite`，数据库文件位于项目根目录：

- `dorm_repair.db`

如果需要重新生成示例数据：

1. 停止后端
2. 删除 `dorm_repair.db`
3. 重新启动后端

后端会根据 `schema.sql` 和 `data.sql` 重新建库并导入数据。

## 图片上传说明

- 上传后的图片会保存在项目根目录的 `uploads` 文件夹
- 学生端可上传故障现场图片
- 管理端维修人员可上传维修结果图片

## 常见问题

### 1. 端口被占用

如果 `8080`、`5173`、`5174` 被占用，先关闭已有进程再启动。

### 2. 首次启动较慢

首次运行前端时需要安装依赖，速度取决于网络情况。

### 3. 示例数据没有变化

如果数据库文件已存在，可能还是旧数据。删除 `dorm_repair.db` 后重新启动后端即可。
